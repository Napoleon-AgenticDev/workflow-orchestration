# Angular Specification Documents

This directory contains comprehensive Angular specification documents derived from the official Angular documentation. These specifications serve as AI-readable context for GitHub Copilot and other coding agents to generate consistent, best-practice Angular implementations.

## Directory Structure

All Angular specifications are now located in this single `angular/` folder:

- `angular-components-templates.specification.md` - Component architecture, templates, and lifecycle
- `angular-forms-validation.specification.md` - Forms handling and validation
- `angular-http-observables.specification.md` - HTTP client and RxJS patterns
- `angular-services-di.specification.md` - Services and dependency injection patterns
- `angular-testing-performance.specification.md` - Testing strategies and performance optimization

## Completed Specifications

- **[Components and Templates](./angular-components-templates.specification.md)**

  - Component lifecycle and architecture
  - Template syntax and data binding
  - Signal-based reactive updates
  - Event handling and user interaction
  - Performance optimization patterns

- **[Services and Dependency Injection](./angular-services-di.specification.md)**

  - Injectable services and DI system
  - Signal-based state management in services
  - Provider configuration and scoping
  - Service testing and mocking patterns
  - Error handling and recovery strategies

- **[Forms and Validation](./angular-forms-validation.specification.md)**

  - Reactive forms with TypeScript type safety
  - Custom validators and async validation
  - Dynamic forms with FormArray
  - Form accessibility and WCAG compliance
  - Testing form components and validation

- **[HTTP Client and Observables](./angular-http-observables.specification.md)**

  - HTTP client service patterns
  - RxJS observable integration
  - Interceptors for cross-cutting concerns
  - Error handling and retry logic
  - Performance optimization for API calls

- **[Testing and Performance](./angular-testing-performance.specification.md)**
  - Unit testing with Jasmine and TestBed
  - Component integration testing
  - Service mocking and dependency injection
  - Performance monitoring and optimization
  - Bundle analysis and lazy loading strategies
